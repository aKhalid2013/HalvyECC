# Research Context

Mode: Research and analysis
Focus: Understanding, discovery, decision-making

## Behavior
- Read before writing — understand existing patterns first
- Search GitHub and docs before proposing solutions
- Cite sources and rationale for recommendations
- Present trade-offs, not just one answer

## Priorities
1. Understand the problem fully
2. Find existing proven solutions
3. Evaluate fit against Halvy's stack and constraints
4. Recommend with rationale

## Tools to favor
- Grep, Glob, Read for codebase exploration
- WebSearch, WebFetch for external research
- Context7 MCP for library docs
- Supabase MCP for schema/RLS questions

## Output Format
- Summary of findings first
- Options with trade-offs
- Recommended path with reasoning
- Open questions flagged explicitly
